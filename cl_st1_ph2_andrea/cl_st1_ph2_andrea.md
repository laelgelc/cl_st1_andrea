# cl-st1_ph2_andrea - Pipeline Programmes

This document summarises the programmes used in the Phase 2 project pipeline. The pipeline prepares the verbal corpus, selects keywords, builds SAS input files, processes post-SAS factor outputs, generates reporting materials, and prepares factor-pole interpretations.

| Step | Programme / Command                | Purpose                                                                                                                                                                 | Main Inputs                                                                       | Main Outputs                                                                                   |
|-----:|------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------|
|    1 | `tag.py`                           | Tags the source corpus and produces token, tag, and lemma files grouped by decade.                                                                                      | Phase corpus text files                                                           | `corpus/07_tagged/<Decade>/`                                                                   |
|    2 | `keylemmas.py`                     | Extracts decade-level key lemmas from the tagged corpus. The `--cutoff` argument controls the minimum threshold for retaining lemmas.                                   | `corpus/07_tagged/`                                                               | `corpus/08_keylemmas/<Decade>.tsv`                                                             |
|    3 | `select_kws_stratified.py`         | Selects a stratified keyword set by decade. The selected keywords are later used to construct binary keyword columns for SAS.                                           | `corpus/08_keylemmas/`                                                            | `corpus/09_kw_selected/keywords.txt`                                                           |
|    4 | `rm -rf columns columns_clean`     | Removes previously generated keyword-column folders before rebuilding them, ensuring that the column matrix matches the latest keyword list.                            | Existing `columns/` and `columns_clean/` directories, if present                  | Clean workspace for regenerated columns                                                        |
|    5 | `columns.py`                       | Builds binary keyword columns for each text based on the final selected keyword list. Also writes file and keyword index mappings.                                      | `corpus/09_kw_selected/keywords.txt`; corpus files                                | `columns/`; `columns_clean/`; `file_ids.txt`; `index_keywords.txt`                             |
|    6 | `merge_columns.py`                 | Merges the per-text keyword columns into the space-separated counts matrix required by the SAS LMDA workflow.                                                           | `columns_clean/`                                                                  | `sas/counts.txt`                                                                               |
|    7 | `sas_formats.py`                   | Generates SAS format and label files that map keyword variable IDs, such as `v000001`, to readable word labels.                                                         | Keyword/index files                                                               | `sas/word_labels_format.sas`; `sas/word_labels_full_format.sas`; other SAS helper format files |
|    8 | SAS workflow                       | Runs the external SAS Lexical Multi-dimensional Analysis workflow.                                                                                                      | `sas/counts.txt`; SAS format files                                                | SAS factor-score, loading, ANOVA, and parameter outputs                                        |
|    9 | `factor_lists.py`                  | Reads SAS factor outputs and creates readable positive and negative loading lists for each factor.                                                                      | SAS factor loading outputs                                                        | `factors/`                                                                                     |
|   10 | `corpus_size.py`                   | Calculates corpus-size summaries for reporting and for checking balance across decades.                                                                                 | Corpus files and/or metadata                                                      | `corpus_size/corpus_size.tsv`                                                                  |
|   11 | `latex_boxplots/latex_boxplots.py` | Generates LaTeX/TikZ boxplots for factor dimensions, including slide/report-ready visualisations.                                                                       | Factor scores and related data                                                    | `latex_boxplots/slides/`                                                                       |
|   12 | `latex_anova_table.py`             | Creates a LaTeX ANOVA summary table for decade effects across factors, including F values, p values, R², and percent R².                                                | SAS ANOVA and parameter outputs                                                   | `latex_tables/anova_decade.tex`                                                                |
|   13 | `examples.py`                      | Selects representative high-scoring texts by factor pole and decade, then writes LaTeX examples with factor-loading lemmas highlighted.                                 | Factor scores; factor loading lists; corpus texts                                 | `examples/`                                                                                    |
|   14 | `score_details.py`                 | Produces a sanity-check report showing, for each text and factor, which positive- and negative-pole loading words are present.                                          | Factor loading lists; corpus texts; scores                                        | `examples/score_details.txt`                                                                   |
|   15 | `examples_txt.py`                  | Generates plain-text versions of the selected examples, including score metadata and loading words. These are useful for manual review and interpretation prompts.      | Selected examples; factor scores; loading words                                   | `examples_txt/`                                                                                |
|   16 | `interpretation_prompts.py`        | Builds one interpretation prompt per factor pole by combining factor loadings, decade mean scores, plain-text examples, and score-details information.                  | `factors/`; `examples_txt/`; `examples/score_details.txt`; factor score summaries | `interpretation/input/`                                                                        |
|   17 | `generate_interpretation_gpt.py`   | Submits interpretation prompts to the configured GPT model and writes one response file per factor pole. Requires `OPENAI_API_KEY` in the environment or in `env/.env`. | `interpretation/input/`                                                           | `interpretation/output/`                                                                       |

## Pipeline Overview

The pipeline can be understood as five broad stages:

| Stage                       | Description                                                                                                          | Programmes                                                                                                                             |
|-----------------------------|----------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| Corpus preparation          | Tag the corpus and identify key lexical items by decade.                                                             | `tag.py`, `keylemmas.py`                                                                                                               |
| Keyword matrix construction | Select keywords, build binary columns, and merge them into the SAS counts matrix.                                    | `select_kws_stratified.py`, `columns.py`, `merge_columns.py`, `sas_formats.py`                                                         |
| SAS-based LMDA              | Run the external SAS workflow to generate factor outputs and statistical results.                                    | SAS workflow                                                                                                                           |
| Post-SAS reporting          | Convert SAS outputs into readable factor lists, corpus summaries, plots, tables, examples, and score-detail reports. | `factor_lists.py`, `corpus_size.py`, `latex_boxplots.py`, `latex_anova_table.py`, `examples.py`, `score_details.py`, `examples_txt.py` |
| Interpretation              | Build factor-pole interpretation prompts and submit them to GPT.                                                     | `interpretation_prompts.py`, `generate_interpretation_gpt.py`                                                                          |

## Notes

- The pipeline should be run from the Phase 2 project directory.
- The SAS step is external and must be completed before the post-SAS programmes can produce factor lists, ANOVA tables, boxplots, and interpretation materials.
- `generate_interpretation_gpt.py` requires an OpenAI API key, either in the system environment or in `env/.env`.

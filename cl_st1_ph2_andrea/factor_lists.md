# Development Specification: `factor_lists.py`

## 1. Programme Purpose

`factor_lists.py` converts SAS rotated factor-analysis output into readable factor-loading lists.

It reads:

- the SAS rotated factor table;
- the keyword index mapping variable IDs to lemmas;

and writes:

1. factor-pole loading lists with lemmas;
2. factor-pole loading lists with variable IDs;
3. primary-loading-only lists.

The programme is used after the SAS LMDA factor-analysis stage and before example extraction and interpretation-prompt generation.

---

## 2. Project Context

The project performs Lexical Multi-Dimensional Analysis (LMDA) on decade-organised commercial subcorpora.

The programme is used in both:

1. **Phase 2: commercial verbal subcorpus**
   - transcript texts of spoken/audio-verbal commercial content;

2. **Phase 3: commercial visual subcorpus**
   - textual descriptions of commercial visual content.

The same logic applies in both phases. The project name is inferred from the current working directory unless explicitly provided.


```text

```

## 3. Inputs

### 3.1 SAS Rotated Output

Default:

```text
sas/output_<project>/rotated.csv
```

This file is generated by the SAS factor-analysis script.

Expected columns include:

```text
_NAME_
loaded
factor
pole
Factor1
Factor2
...
```

### 3.2 Required `rotated.csv` Columns

| Column | Description |
|---|---|
| `_NAME_` | Variable ID, e.g. `v000001` |
| `loaded` | Binary indicator showing whether variable has a primary loading |
| `factor` | Primary factor assignment, e.g. `fac1` |
| `pole` | Primary pole assignment, `1` or `-1` |
| `Factor<n>` | Loading value for factor dimension `<n>` |

At least one column matching:

```text
Factor1
Factor2
...
```

must exist.

### 3.3 Keyword Index File

Default:

```text
index_keywords.txt
```

Expected format:

- no header;
- space-separated;
- two columns:

```text
keyword_id lemma
```

Example:

```text
000001 camera
000002 product
000003 screen
```

The programme converts keyword IDs to variable IDs by prefixing `v`:

```text
000001 → v000001
```

```text
sas/output_<project>
```

```text
--index-file
```

Default:


```text
--output-dir
```


## 4. Outputs

The programme writes output under:

```text
factors/
```

or the directory specified by `--output-dir`.

### 4.1 Full Word Loading Lists

Path pattern:

```text
factors/f<n>_<pole>.txt
```

Examples:

```text
factors/f1_pos.txt
factors/f1_neg.txt
factors/f2_pos.txt
factors/f2_neg.txt
```

These files contain:

- primary loadings;
- secondary loadings.

Primary loadings are written without outer parentheses:

```text
camera (.62)
```

Secondary loadings are written inside outer parentheses:

```text
(product (.34))
```

First line:

```text
variables loading on this pole = <count>
```

Second line:

```text
comma-separated loading entries
```

### 4.2 Variable-ID Loading Lists

Path pattern:

```text
factors/var_id/f<n>_<pole>_var_id.txt
```

Examples:

```text
factors/var_id/f1_pos_var_id.txt
factors/var_id/f1_neg_var_id.txt
```

Same format as the full word lists, but with variable IDs instead of lemmas:

```text
v000001 (.62), (v000002 (.34))
```

### 4.3 Primary-Only Word Loading Lists

Path pattern:

```text
factors/primary_loadings/f<n>_<pole>.txt
```

Examples:

```text
factors/primary_loadings/f1_pos.txt
factors/primary_loadings/f1_neg.txt
```

These files contain only primary loadings.

Secondary loadings are excluded.

### 4.4 Output Encoding

All output files must be:

```text
UTF-8
```

## 5. Factor Loading Semantics

### 5.1 Primary Loading

A variable’s primary loading is determined by SAS and recorded in `rotated.csv` using:

```text
loaded = 1
factor = fac<n>
pole = 1 or -1
```

Rows where `loaded != 1`, `factor` is missing, or `pole` is missing must be skipped.

### 5.2 Primary Pole

If:

```text
pole = 1
```

the variable belongs to the positive pole.

If:

```text
pole = -1
```

the variable belongs to the negative pole.

### 5.3 Primary Factor Column

The factor label in `factor` is mapped to the corresponding loading column:

```text
fac1 → Factor1
fac2 → Factor2
```

The loading value in that column is used as the primary loading score.

### 5.4 Secondary Loadings

For every factor loading column other than the primary factor column:

1. read its loading value;
2. skip missing values;
3. if absolute value is greater than or equal to `--cutoff`, treat it as a secondary loading.

If the secondary loading value is positive:

```text
f<n>_pos
```

If negative:

```text
f<n>_neg
```

Secondary loading entries must be enclosed in outer parentheses.

Example:

```text
(product (.34))
```

---

## 6. Formatting Requirements

### 6.1 Loading Value Format

Loading values must be rounded to two decimals.

Leading zero must be removed.

Examples:

| Value | Formatted |
|---:|---|
| `0.45` | `.45` |
| `-0.45` | `-.45` |
| `1.12` | `1.12` |

### 6.2 Sorting

Within each output file, entries must be sorted by descending absolute loading value.

Example order:

```text
camera (.72), product (.61), screen (.44)
```

### 6.3 Count Line

The first line must report the total number of entries written to that file:

```text
variables loading on this pole = 12
```

For `factors/primary_loadings`, the count must reflect only primary entries.

For `factors/` and `factors/var_id`, the count must include primary and secondary entries.

### 6.4 Entry Conventions

Primary word entry:

```text
lemma (.45)
```

Secondary word entry:

```text
(lemma (.45))
```

Primary variable-ID entry:

```text
v000001 (.45)
```

Secondary variable-ID entry:

```text
(v000001 (.45))
```

## 7. Functional Workflow

### Step 1: Parse Arguments

Read:

```text
--project
--sas-output-dir
--index-file
--output-dir
--cutoff
```

Validate:

```text
--cutoff >= 0
```

### Step 2: Resolve Paths

If `--sas-output-dir` is omitted, set:

```text
sas/output_<project>
```

Set:

```text
rotated_path = sas_output_dir / "rotated.csv"
```

### Step 3: Load `rotated.csv`

The programme must:

1. validate that `rotated.csv` exists;
2. load it as a CSV using pandas;
3. verify required columns:
   - `_NAME_`
   - `loaded`
   - `factor`
   - `pole`
4. verify at least one `Factor<n>` column exists.

### Step 4: Load Keyword Index

The programme must:

1. validate that `index_keywords.txt` exists;
2. read it as a headerless file;
3. split each line using `split(maxsplit=1)`;
4. validate keyword IDs are numeric;
5. create mapping:

```text
v<keyword_id> → lemma
```

Example:

```text
v000001 → camera
```

### Step 5: Iterate Over Rotated Rows

For each row in `rotated.csv`:

1. get variable ID from `_NAME_`;
2. map variable ID to lemma using index;
3. skip row if:
   - `loaded != 1`;
   - `factor` is missing;
   - `pole` is missing.
4. determine primary factor and pole;
5. write primary loading entry to the appropriate result list;
6. inspect all other factor columns for secondary loadings;
7. write secondary loading entries where `abs(value) >= cutoff`.

### Step 6: Write Outputs

Create:

```text
factors/
factors/var_id/
factors/primary_loadings/
```

Write all result files.

### Step 7: Print Summary

Print:

- project name;
- rotated input path;
- keyword index path;
- output directory;
- number of factor-list files written.

## 8. Error Handling Requirements

### 8.1 Missing Rotated File

Raise:

```text
FileNotFoundError
```

if:

```text
sas/output_<project>/rotated.csv
```

does not exist.

### 8.2 Missing Required Columns

Raise:

```text
ValueError
```

if `rotated.csv` lacks any required column.

### 8.3 No Factor Columns

Raise:

```text
ValueError
```

if no columns beginning with `Factor` are found.

### 8.4 Missing Keyword Index

Raise:

```text
FileNotFoundError
```

if `index_keywords.txt` does not exist.

### 8.5 Headered Keyword Index

Raise:

```text
ValueError
```

if `index_keywords.txt` appears to contain a header.

### 8.6 Invalid Keyword ID

Raise:

```text
ValueError
```

if a keyword ID is not numeric.

### 8.7 Empty Keyword Index

Raise:

```text
ValueError
```

if no keyword entries are found.

### 8.8 Negative Cutoff

Raise:

```text
ValueError
```

if `--cutoff < 0`.

### 8.9 No Loading Entries Generated

Raise:

```text
ValueError
```

if no factor-loading entries are generated.

This may indicate:

- empty or invalid `rotated.csv`;
- no rows with `loaded = 1`;
- mismatch between SAS output and expected structure.

## 9. Non-Functional Requirements

### 9.1 Encoding

Output files must be UTF-8.

### 9.2 Determinism

The programme must be deterministic:

- input rows processed in file order;
- entries sorted by descending absolute loading value;
- output filenames deterministic.

### 9.3 No Input Mutation

The programme must not modify:

```text
rotated.csv
index_keywords.txt
```

### 9.4 Output Overwrite Behaviour

The programme may overwrite existing files under:

```text
factors/
factors/var_id/
factors/primary_loadings/
```

### 9.5 No Header Changes

Output files intentionally include the first summary line:

```text
variables loading on this pole = <count>
```

This line must be preserved because downstream scripts skip or interpret it.

### 9.6 Compatibility with `score_details.py`

The `factors/var_id/*.txt` files must preserve the primary-versus-secondary convention:

- primary IDs outside parentheses;
- secondary IDs inside parentheses.

`score_details.py` uses this convention to identify primary loading words.

## 10. Downstream Dependencies

The outputs are consumed by:

### 10.1 `examples.py`

Uses:

```text
factors/f<n>_pos.txt
factors/f<n>_neg.txt
```

to identify primary lemmas for bolding in LaTeX examples.

### 10.2 `score_details.py`

Uses:

```text
factors/var_id/f<n>_pos_var_id.txt
factors/var_id/f<n>_neg_var_id.txt
```

to identify which primary loading variables are present in each text.

### 10.3 `interpretation_prompts.py`

Uses:

```text
factors/f<n>_pos.txt
factors/f<n>_neg.txt
```

as the factor-loading section in LLM interpretation prompts.

Therefore, the output filenames and primary/secondary formatting conventions must remain stable.

---

## 11. Acceptance Criteria

The programme is correct if:

1. It infers project name from current working directory by default.
2. It reads `sas/output_<project>/rotated.csv`.
3. It reads `index_keywords.txt`.
4. It validates required input structure.
5. It generates readable word-based factor lists.
6. It generates variable-ID factor lists.
7. It generates primary-only word lists.
8. It writes files under `factors/`.
9. It preserves primary/secondary loading notation.
10. It formats loadings with two decimals and no leading zero.
11. It sorts entries by descending absolute loading.
12. `examples.py`, `score_details.py`, and `interpretation_prompts.py` can use the outputs without modification.

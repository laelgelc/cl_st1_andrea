# Development Specification: `merge_columns.py`

## 1. Programme Purpose

`merge_columns.py` merges the binary keyword-presence column files generated by `columns.py` into a single space-separated matrix file for SAS factor analysis.

The programme reads:

- full metadata rows from `columns/000001.txt`;
- binary keyword columns from `columns_clean/*.txt`;

and writes:

```text
sas/counts.txt
```

The resulting file is the main input matrix for the SAS Lexical Multi-Dimensional Analysis workflow.

---

## 2. Project Context

The project performs LMDA on decade-organised commercial subcorpora.

The same programme is used in:

1. **Phase 2: commercial verbal subcorpus**
   - transcript texts of spoken/audio-verbal commercial content;

2. **Phase 3: commercial visual subcorpus**
   - textual descriptions of visual commercial content.

In both phases, `columns.py` creates:

```text
columns/
columns_clean/
file_ids.txt
index_keywords.txt
```

`merge_columns.py` then combines these column files into:

```text
sas/counts.txt
```

which is consumed by the SAS factor-analysis script.

---

## 3. Inputs

### 3.1 Full Column Directory

Default:

```text
columns
```

This directory contains one full column file per keyword:

```text
columns/000001.txt
columns/000002.txt
...
```

### 3.2 Reference Full Column File

The programme uses:

```text
columns/000001.txt
```

as the reference file.

Expected format:

- no header;
- space-separated;
- three fields per row:

```text
file_id decade presence
```

Example:

```text
t000001 1950 0
t000002 1950 1
```

The programme uses only the first two fields:

```text
file_id decade
```

The final `presence` field in the reference file is removed before merging.

### 3.3 Clean Column Directory

Default:

```text
columns_clean
```

This directory contains one clean binary column file per keyword:

```text
columns_clean/000001.txt
columns_clean/000002.txt
...
```

Expected format:

- no header;
- first line = keyword ID;
- all subsequent lines = binary flags.

Example:

```text
000001
0
1
0
```

### 3.4 Clean Column File Ordering

The programme must read clean column files in sorted filename order.

Example:

```text
000001.txt
000002.txt
000003.txt
...
```

This order determines the keyword column order in `sas/counts.txt`.

---

## 4. Output

### 4.1 Output Directory

Default:

```text
sas
```

The programme must create it if it does not exist.

### 4.2 Output File

Default:

```text
sas/counts.txt
```

Format:

- no header;
- space-separated;
- each row corresponds to one text/commercial;
- columns are:

```text
file_id decade kw000001 kw000002 kw000003 ...
```

The keyword columns themselves are unnamed in the file; SAS assigns names when reading.

Example:

```text
t000001 1950 0 1 0 0
t000002 1950 1 0 0 1
```

### 4.3 Output Delimiter

The output must use spaces, not tabs.

### 4.4 Output Header

The output must not include a header.

---

## 5. Functional Requirements

### 5.1 Create Output Directory

The programme must ensure the output directory exists:

```text
sas/
```

### 5.2 Read Reference File

The programme must read:

```text
columns/000001.txt
```

and validate:

1. file exists;
2. file is not empty;
3. each non-empty line has exactly three fields.

Expected row format:

```text
file_id decade presence
```

Example:

```text
t000001 1950 0
```

### 5.3 Build Initial Rows

For each line in the reference file, keep only:

```text
file_id decade
```

Discard the reference keyword’s presence value.

Example:

Input:

```text
t000001 1950 0
```

Initial row:

```text
["t000001", "1950"]
```

### 5.4 Discover Clean Column Files

The programme must collect all files matching:

```text
columns_clean/*.txt
```

and sort them.

If no clean column files are found, raise:

```text
FileNotFoundError
```

### 5.5 Validate Each Clean Column File

For each clean column file:

1. Read all non-empty lines.
2. Validate file is not empty.
3. Validate first line equals the file stem.

Example:

File:

```text
columns_clean/000001.txt
```

First line must be:

```text
000001
```

4. Treat all remaining lines as binary flags.
5. Validate number of flags equals number of reference rows.
6. Validate each flag is exactly:

```text
0
```

or:

```text
1
```

### 5.6 Merge Binary Flags

For each clean column file, append the binary flag from each row to the corresponding initial row.

Example:

Initial row:

```text
["t000001", "1950"]
```

After first keyword:

```text
["t000001", "1950", "0"]
```

After second keyword:

```text
["t000001", "1950", "0", "1"]
```

### 5.7 Write Merged Matrix

Write all merged rows to:

```text
sas/counts.txt
```

Each row must be written as:

```python
" ".join(row)
```

No header.

## 6. File Format Contracts

These format contracts must not be changed without updating downstream SAS code.

### 6.1 `columns/000001.txt`

Required format:

```text
file_id decade presence
```

No header.

Space-separated.

### 6.2 `columns_clean/<Keyword ID>.txt`

Required format:

```text
keyword_id
presence
presence
presence
...
```

No header.

### 6.3 `sas/counts.txt`

Required format:

```text
file_id decade v000001 v000002 v000003 ...
```

No header.

Space-separated.

The output keyword values are positional. The first keyword value corresponds to:

```text
v000001
```

the second to:

```text
v000002
```

and so on.

## 7. Error Handling Requirements

### 7.1 Missing Reference File

If `columns/000001.txt` does not exist, raise:

```text
FileNotFoundError
```

with message:

```text
Reference file not found: columns/000001.txt
```

### 7.2 Empty Reference File

If the reference file contains no non-empty lines, raise:

```text
ValueError
```

### 7.3 Invalid Reference Row Format

If any reference row does not contain exactly three fields, raise:

```text
ValueError
```

Message should indicate:

- file path;
- line number;
- expected field count;
- actual field count.

### 7.4 Missing Clean Column Files

If no clean column files are found, raise:

```text
FileNotFoundError
```

### 7.5 Empty Clean Column File

If a clean column file contains no non-empty lines, raise:

```text
ValueError
```

### 7.6 Keyword ID Mismatch

If the first line of a clean column file does not match its filename stem, raise:

```text
ValueError
```

Example:

```text
Keyword ID mismatch in 000001.txt: first line is 000002, expected 000001
```

### 7.7 Row Count Mismatch

If the number of binary flags in a clean column file does not match the number of reference rows, raise:

```text
ValueError
```

### 7.8 Invalid Binary Flag

If any binary flag is not exactly `0` or `1`, raise:

```text
ValueError
```

## 8. Non-Functional Requirements

### 8.1 Encoding

All files must be read and written using:

```text
UTF-8
```

### 8.2 Determinism

The programme must produce deterministic output:

- clean column files sorted lexicographically;
- row order inherited from `columns/000001.txt`;
- keyword column order inherited from sorted clean filenames.

### 8.3 No Header Injection

The programme must not add a header to:

```text
sas/counts.txt
```

### 8.4 Space-Separated Output

The output must use spaces, not tabs.

### 8.5 No Data Transformation

The programme must not change binary values.

It only merges them.

### 8.6 No Corpus Mutation

The programme must not modify:

```text
columns/
columns_clean/
file_ids.txt
index_keywords.txt
corpus/
```

It only reads column files and writes `sas/counts.txt`.

---

## 9. Downstream Dependencies

The output `sas/counts.txt` is consumed by SAS scripts.

The SAS script expects:

```text
filename decade v000001-v<lastkeyword>
```

The first two columns are metadata:

1. `filename`
2. `decade`

The remaining columns are binary keyword variables in sequential order.

If `sas/counts.txt` includes a header, has tabs instead of spaces, or has mismatched row counts, the SAS factor analysis may fail or produce incorrect results.

---

## 10. Acceptance Criteria

The programme is correct if:

1. It reads `columns/000001.txt`.
2. It verifies that every row in `columns/000001.txt` has three fields.
3. It keeps only `file_id` and `decade` from the reference file.
4. It reads all `columns_clean/*.txt` files in sorted order.
5. It verifies each clean file’s keyword ID line.
6. It verifies row counts match across all clean files.
7. It verifies all binary flags are `0` or `1`.
8. It writes `sas/counts.txt`.
9. `sas/counts.txt` has no header.
10. `sas/counts.txt` is space-separated.
11. Each output row has:

```text
2 + number_of_keyword_files
```

fields.

12. The SAS script can read the output successfully.

## 13. Summary

`merge_columns.py` constructs the final binary keyword-presence matrix for SAS.

Its core responsibility is:

```text
Merge text metadata from columns/000001.txt with all clean binary keyword columns into sas/counts.txt.
```

It must preserve:

- row order;
- keyword column order;
- binary values;
- space-separated output;
- no-header format;
- SAS compatibility.
